Meet Dashboard, an at-a-glance view of what's happening at DoSomething.org:

Members

It's one of our big goals, so it deserves to be front and center. Keep an eye on progress towards our end-of-year membership goal as it happens.

Weather

Ever forgotten to bring your umbrella when heading out to get lunch? Dashboard gives you the current conditions & crazy-accurate rain forecasts for the next hour, powered by Dark Sky.

Social Status

We're pretty popular on the internet. Keep tabs on how many likes and followers we have, and see the latest @dosomething mentions as they come in via the ticker at the bottom of the screen.

Conference Rooms

Need a place to talk business, but not sure what rooms are free? Dashboard will tell you what conference rooms & phone booths are free (and for how long). Check their full booking schedule with a single click.

Tech Status

Dashboard provides a quick overview of the tech team's antics: how long since code was last deployed to the website, how many commits are waiting to go live, and progress in the current sprint.

Shortcuts

It can be a pain to remember the URLs to all the different services we use. Dashboard acts as your digital chauffeur by giving you easy shortcuts. Visit do/ in your browser for a list of services, or just add a keyword to the URL to jump directly to the associated service (for example, do/bugs.)

And finally, let's do something fun…

Introducing DS Play, the office DJ. We listen to music all the time in the office, but it can be a hassle to do so together. Now everyone can share their favorite songs and tune in live to what the rest of the office is listening to.

Check it out at do/dashboard!

Technical Notes (for the curious):

Dashboard was built using a lot of really cool technology. The server runs Node.js, with help from the Express.js webapp framework for routing and session handling. Socket.io provides whizz-bang cross-browser WebSockets magic to broadcast real-time updates from the server. Most data is collected by polling various REST APIs, and cached on the server. Cached data is sent directly to individual clients when they connect, and changes are broadcast to all connected clients when they occur.

Member statistics are calculated every 10 minutes by doing some estimates. It's not perfect, but it's pretty darn accurate and significantly more feasible to automate. We take the total number of email subscriptions from the fabulous Mailchimp API, and add that to the number of subscribers on Mobile Commons (cut down to 75% to account for duplicate accounts). We also add in 120,000 users from the Bully campaign.

Weather is provided by the (magical) Dark Sky API, which gives short-term forecasts with precise details for precipitation over the next hour.

Conference room availability is provided by the Google Calendar v3 API. Free/busy information is parsed server-side to find any contiguous events before sending data off to clients.

Commit status is provided by the Github API. Code deploys are tracked by pinging the Dashboard server with a POST request when Chris's deploy script is run. Sprint status is obtained using the Redmine REST API.

Inspired by Github Play, DS Play is a custom-built playlist manager and API written in NodeJS, which handles queueing songs & communicating with a desktop Rdio client via AppleScript. The audio from the desktop Rdio client is streamed across the office using Nicecast. Because all interaction with the DS Play server occur through a standard REST API, clients could be built for other platforms and devices in the future with minimal changes to the server. The code for DS Play will be open-sourced on Github so that other companies can use and improve it.

The Twitter ticker is powered by the Twitter Streaming API, with a filter set to subscribe to @dosomething mentions. This allows Dashboard to receive instant updates without the overhead of constantly querying yet another REST API.

Social icons are from the wonderful Symbolset by Oak Studios. Superhero icons used for the conference rooms are from their respective source material. Most other icons were created specifically for Dashboard, and will be released as part of a larger set sometime in the future.
